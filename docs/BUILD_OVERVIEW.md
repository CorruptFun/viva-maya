## 1. Overview & tech stack

**Viva Ton** is a casino-themed match-3 PWA — the "swap adjacent pieces, clear
runs of 3+, chain cascades, earn special pieces" board loop dressed as a modern
slot screen, with a light lives/energy pool, a daily bonus spin, and a weekly
endless race wrapped around it. Mobile-first, installable, fully offline.

| | |
|---|---|
| **Engine** | Phaser 3.90 |
| **Build** | Vite 7 + `vite-plugin-pwa` 1.x (Workbox `generateSW`, `autoUpdate`) |
| **Language** | TypeScript ~5.8, `strict` + `noUnusedLocals`/`noUnusedParameters`/`noFallthroughCasesInSwitch` |
| **Assets** | **Zero binary game assets** — all symbol art is system emoji rendered into textures at boot; all SFX are synthesized procedurally in WebAudio; icons are generated by a script |
| **Persistence** | `localStorage` (saves + mute + swap-sound preference) |
| **Target** | Portrait 720×1280 design canvas, `Scale.FIT` + `CENTER_BOTH`, `standalone` PWA |
| **Live URL** | https://corruptfun.github.io/viva-ton/ |
| **License** | **Proprietary — © 2026 CorruptFun LLC. All rights reserved.** Public repo for reference only; no license granted (`LICENSE`, `package.json` `"license": "UNLICENSED"`) |

**Zero-assets approach.** `public/` holds only PWA/favicon/OG images (themselves
script-generated by `scripts/gen-icons.mjs`). Nothing in the *game* is a loaded
asset: `createAllTextures` (boot) renders emoji glyphs and hand-composed graphics
(the 7, BAR pill, jackpot disc, chips, cards, bulbs, sparks, rings, sweeps) into
`DynamicTexture`s; special-piece overlays are composed lazily on first use
(`ensurePieceTexture`); every sound in `src/audio/sfx.ts` is built at runtime from
oscillators, noise buffers, filters and gain envelopes. This keeps the PWA small
and fully functional offline.

## 2. Architecture & file map

### The layering rule

```
src/core/     PURE logic — no Phaser imports. Owns "truth" (board state, rules, save).
src/config.ts Feel/tuning constants (sizes, timings, points, lives, colors).
src/scenes/   Phaser scenes — input, tweens, choreography, HUD. Renders core state.
src/view/     Shared rendering helpers — textures, ambient background, UI widgets.
src/audio/    Procedural WebAudio SFX singleton.
```

The `src/core/` modules import only each other and `../config` (which imports only a
type). They never touch Phaser, so the rules are unit-testable and deterministic. The
scenes own all sprites/tweens/DOM and treat the core as a model: e.g. `GameScene`
calls `board.matchWave()`/`applyGravity()`/`refill()` and animates the returned
data (`ClearWave`, `FallMove[]`, `Spawn[]`, `BlastEvent[]`). Tuning happens in
`config.ts` without touching logic or rendering.

## 3. Features & systems

### Board & matching — `src/core/board.ts`, `src/config.ts`
8×8 grid (`CELL=80`, board at `BOARD_X=40`, `BOARD_Y=300`). Six symbols
(`cherry 🍒`, `seven` styled 7, `diamond 💎`, `bell 🔔`, `clover 🍀`, `bar` navy pill);
early levels use the first 5, level 4+ all 6. Swap adjacent pieces via swipe (drag
≥ `0.3·CELL`) or tap-select → tap-adjacent. A swap is valid only if it makes a run
≥3 **or** activates a special; invalid swaps thud and snap back with **no move
consumed**. Board generation never spawns pre-existing matches and guarantees ≥1
valid move; a dead board reshuffles (`regenerate`). `findFirstValidMove` doubles as
the hint/autoplay engine.

### Special pieces & combos — `src/core/board.ts`
Created at the swapped cell when possible, else run intersection, else run middle.
Match-created specials are blast-protected during their birth wave.

| Match shape | Piece | Effect |
|---|---|---|
| 4 in a row | Wild Reel (chevrons) | Clears the line **perpendicular** to the match |
| L / T (crossing runs) | Dice Bomb (🎲 badge) | 3×3 blast |
| 5+ straight | Jackpot Chip (gold 🎰 disc, colorless) | Color bomb — clears all of one color |

Any blast hitting a special chains it (`chainExpand`); a jackpot hit by a blast
takes a **random present color**. Swap-combos (epicenter = drop cell): Reel+Reel →
full cross; Bomb+Bomb → 5×5; Reel+Bomb → 3 rows + 3 cols; Jackpot+normal → that
color; Jackpot+Reel/Bomb → converts every piece of that color into the special then
detonates all; Jackpot+Jackpot → whole board.

### Scoring & cascades — `GameScene.playWave` / `addScore`
`cleared.length × POINTS_PER_PIECE(20) × cascade` per wave (wave 1 ×1, wave 2 ×2,
…). `doubleScore` boost multiplies everything ×2 via `scoreMult`. `COMBO ×n` popup at
cascade ≥2; **MEGA WIN** at ≥4 (siren + big vibrate + cabinet flash). Win adds
`+60 × unused moves`. `BEST` = highest single score, persisted.

### Levels — `src/core/levels.ts`, `LevelSelectScene`
`LEVEL_COUNT=100`. `levelSpec(n)` is deterministic (seed `0xC0FFEE ^ n·2654435761`):
same goals every attempt, random board per attempt.
- **Objectives:** collect N of 1 symbol (L1–2), 2 (L3–7), 3 (L8+); each
  `N = min(45, 10 + round(2.2n))`.
- **Moves:** `max(14, 26 − floor(n/2)) + 2·objectiveCount`, `+4` breather every 5th
  level.
- **Stars:** by remaining-moves fraction — ≥50% → 3★, ≥25% → 2★, else 1★.
- **Level Select:** masked, drag-scrollable 5-wide grid, auto-scrolled so the current
  level sits mid-viewport; taps are suppressed once a press travels (scroll vs. tap).

### Stars & 10-level milestones — `GameScene.finishWin` / `milestoneSplash`
`recordResult` persists best-of stars and unlocks `n+1`. Clearing any level where
`n % 10 === 0` plays a full-screen "LEVEL n! · ★ N STARS EARNED" splash (heart shower
+ fanfare + jackpot strike) before the calm result card.

### Lives / energy — `src/core/lives.ts`, `GameScene` gate
Generous pool: `LIVES_MAX=10`, one life every `LIFE_REGEN_MS=8min` of **wall-clock**
time (device clock trusted). Only a **loss** drains a life; a mid-level quit after ≥1
move also drains one (`exitToLevels`), closing the quit-to-dodge-loss exploit —
**wins are free.** Regen is banked on every read (`refreshLives`) and persisted only
when it actually changes. Storage: `save.lives` + `save.livesAnchor` (epoch ms the
current regen cycle started; 0 when full). Numbered levels gate on entry (0 lives →
`showLivesGate`: "TAKE A BREAK", faded hearts, live "next life / full" countdown, PLAY
appears when one regenerates); the gate is checked **before** boosts are consumed so a
gated entry never wastes a pending boost. **Endless is never gated.** The v5→v6 save
upgrade does a one-time **grace refill to full** (see [§4](#4-save-schema--migrations)).

### Daily bonus spin — `src/core/daily.ts`, `DailyBonusScene`
One spin per **local calendar day** (`lastSpinDate` `YYYY-MM-DD`). A 3-reel slot that
**always** lands 3-of-a-kind of the prize (gift, not gambling). Prize + streak are
computed and persisted in `performSpin` **before** the animation, so closing the app
mid-celebration loses nothing. Weighted prize table:

| Prize | `BoostType` | Weight |
|---|---|---|
| Wild Reel | `wildReel` | 30 |
| Dice Bomb | `diceBomb` | 25 |
| +5 Moves | `extraMoves` | 20 |
| Double Score | `doubleScore` | 15 |
| Jackpot Chip | `jackpot` | 10 |

Streak = consecutive days (`+1` if yesterday spun, else reset to 1); every 5th streak
day pays **two** prizes. Prizes land in `save.pendingBoosts` and are consumed on the
next **numbered** level start (win or lose; endless never consumes) by
`GameScene.applyBoosts` — specials are planted at random cells in rows 3–7, `+5` moves
each, and `doubleScore` sets `scoreMult=2`. A self-sizing gold banner announces them
at level start.

### Streak flame — `ui.ts addStreakBadge`
Home shows a 🔥 flame pill "N DAY STREAK" when `streak > 0` (the flame is a separate
text object — `letterSpacing` splits emoji surrogate pairs in Phaser's renderer).

### Endless weekly race — `src/core/endless.ts`, `GameScene` endless mode
Unlocks after **Level 30** (`endlessUnlocked` = `save.unlocked > ENDLESS_UNLOCK_LEVEL`,
`ENDLESS_UNLOCK_LEVEL=30`). Entry via the rose ENDLESS pill on Home and Level Select.
Everyone in the same calendar week plays the **same** board (`seedForWeek` FNV-1a →
`mulberry32`); a fixed budget (`ENDLESS_MOVES=30`), all 6 symbols, **no objectives, no
boosts** (planting would change the shared board and break the race). Ends only on
moves-out (`finishEndless`). `recordEndless` persists `endlessBest` per week key
(captured at board creation, so a run crossing the local week boundary is still
attributed correctly), resets on week rollover, and also updates all-time `save.best`.
End card shows **NEW BEST!** / **TIME'S UP**.

### Slot-cabinet visuals — `GameScene.buildBackdrop`/`buildCabinet`, `view/background.ts`
Board sits in a cream cabinet with a gold bezel and a rose "screen is on" glow behind
it. A ring of alternating red/gold marquee **bulbs** runs a traveling chase around the
bezel (outside the 8×8 grid). `addCasinoBackdrop` fills the margins with a gradient
wash, twinkling marquee dot strips, corner bokeh, faint ♥♦♣♠ watermarks, and drifting
motes/tokens (the `game` variant stays calm; motes are menu-only).

### Win-sequence celebration — `GameScene.runWinSequence` and helpers
A multi-beat celebration over the still-visible board, then the animated result card:
- **Beat 0** `celebrateBoard` — cabinet light flash + chip/card token burst.
- **Beat 1** warm bloom + cream camera flash + reel sweep.
- **Beat 2** `winWordmark` — a gold-bezel lozenge stamping the rank word (NICE / GREAT
  / PERFECT) with the single win fanfare.
- **Beat 3** `winFireworks` — staggered spark bursts, capped confetti rain, heart puff
  (skipped under `prefers-reduced-motion`).
- **Beat 4** `buildWinCard` + `buildCoinPayout` — elastic card entrance, star row with
  ascending dings, and a **coin roll-up payout** (chips fly onto a pile as a counter
  rolls `0→chipReward`, where `chipReward = stars·8 + movesLeft·2`, cosmetic only).

The whole sequence is **tap-to-skip** (`overlaySettle` fast-forwards to the settled
card). Milestone levels route through `milestoneSplash` instead, then show the calm
card. *(This celebration is fully present in the committed source and compiles; per
the task brief it may have landed very recently.)*

### Swap-sound picker — `src/audio/sfx.ts`, `ui.ts openSoundPanel`
The "move a piece" sound is selectable: **Silk** (default), **Chime**, **Aurora**,
**Classic**. A ♪ chip on Home opens a MOVE SOUND panel with one pill per sound; tapping
auditions it (plays through mute) and persists the choice (`viva-ton:swapSound`).

### How-to-play / FAQ — `ui.ts openHelpPanel`, `addHelpChip`
A "?" chip opens a scrim + tall card of icon/title/blurb sections (goal, moving,
power-ups, lives, daily bonus, stars, endless), a GOT IT button, tap-outside-to-close,
and a copyright line.

## 4. Save schema & migrations

`localStorage` key **`viva-ton:v1`**; current schema version **`v: 6`**. All access is
`try/catch` wrapped — storage failures (private mode, full) never crash the game; the
loader is shape-tolerant and defaults any missing field.

```ts
interface SaveData {
  v: 6
  best: number                 // highest single score (levels + endless feed in)
  unlocked: number             // highest level attemptable (1-based)
  stars: Record<number, number> // best stars per completed level (1–3)
  lastSpinDate: string | null  // 'YYYY-MM-DD' of last daily spin
  streak: number               // consecutive-day spin streak
  pendingBoosts: BoostType[]   // prizes queued for the next numbered level
  endlessWeek: string | null   // week key the endless best belongs to
  endlessBest: number          // best endless score for endlessWeek
  lives: number                // current lives (0..LIVES_MAX)
  livesAnchor: number          // epoch ms the current regen cycle started (0 = full)
}
```

Separate keys: **`viva-ton:muted`** (`'1'`/`'0'`) and **`viva-ton:swapSound`**
(`silk`/`chime`/`aurora`/`classic`).

**Migration history** (each version adds fields; old saves default the new ones):

| Version | Added |
|---|---|
| v1 | `best` |
| v2 | `unlocked`, `stars` |
| v3 | daily spin (`lastSpinDate`, `streak`, `pendingBoosts`) |
| v4 | endless race (`endlessWeek`, `endlessBest`) |
| v5 | lives/energy (`lives`, `livesAnchor`) — pre-v5 saves start full |
| **v6** | **Grace refill** — the pool grew (3→10) and the break shortened; any save with stored `v < 6` is topped up to full (`lives = LIVES_MAX`, `livesAnchor = 0`) on load so no one is stranded at the old, stingier count |

## 5. Audio

Procedural WebAudio, **zero audio assets** (`src/audio/sfx.ts`). A single `Sfx`
instance with a lazily-created `AudioContext`, a master gain of `0.5` into a
`DynamicsCompressor` (gentle limiter so stacked cascade voices don't clip). The
context is unlocked on the first `pointerdown` (iOS/Chrome autoplay policy) via a
one-time capture-phase listener. Every call is fire-and-forget and swallows errors —
if audio is unavailable the game simply runs silent. Mute state persists; a shared
noise buffer is reused across effects.

**SFX map:**

| Sound | Trigger |
|---|---|
| `uiTap` | Button taps |
| `swap` → `swapSilk`/`swapChime`/`swapAurora`/`swapClassic` | Piece swap (selectable move sound) |
| `invalidThud` | Invalid swap snap-back |
| `pop(cascade)` | Per clear wave — rises one semitone per cascade (`2^((c−1)/12)`) |
| `reelSweep` | Wild Reel line clears + daily reel spin |
| `bombBoom` (+30 ms vibrate) | Dice Bomb detonation |
| `jackpotStrike` | Jackpot fire / MEGA WIN / 3★ / milestone / endless record |
| `starDing(i)` | Ascending ding per earned win star |
| `winFanfare` | Win / milestone / spin celebration |
| `coinCount` | Result-card coin roll-up payout |
| `loseWah` | Out-of-moves loss |
| `reshuffleSwirl` | Board reshuffle |

Mute is toggled via `addMuteChip` (🔊/🔇). The swap sound is chosen in the picker
(`previewSwap` auditions through mute; `setSwapSound` persists).

## 6. Dev / test knobs & verification

All gated behind `import.meta.env.DEV` (stripped from production). Appended to the URL:

| Param | Effect |
|---|---|
| `?level=N` | Boot straight into level N |
| `?endless=1` | Boot the weekly endless race |
| `?lives=N` | Force the life pool to N (test the gate) |
| `?scene=daily\|home\|levelselect` | Boot a specific scene |
| `?auto=MS` | Autoplay hinted moves every MS |
| `?turbo=N` | Scale tween/timer clocks by N (embedded panes starve the RAF clock) |
| `?goal=N` | Override each objective count |
| `?moves=N` | Override the move budget |
| `?plant=1` | Seed specials (reel/bomb/jackpot) near bottom-left |
| `?spin=1` | Force the daily spin available |
| `?autospin=1` | Auto-trigger the spin |
| `?help` / `?sound` | Auto-open the help / sound panel (Home) |

**Verification harness.** In DEV, `GameScene` mirrors model state two ways: an
on-screen **DEV strip** (top-left, `#dbg`) and `document.body.dataset.vegas` (JSON of
level/state/moves/score/objectives/hint), refreshed on a 300 ms timer. Because an
embedded/hidden browser pane starves the RAF clock and drops clicks, screenshots +
the strip/dataset (driven by `?auto`/`?autospin`/`?turbo`) are the reliable
verification channel; real tap-targets are confirmed on-device. `main.ts` also renders
an on-screen error surface (`#err`) for `error`/`unhandledrejection` in DEV.

## 7. Build & deploy

**npm scripts** (`package.json`):

| Script | Command | Purpose |
|---|---|---|
| `dev` | `vite` | Dev server (`:5173`, strict port) |
| `build` | `tsc && vite build` | Typecheck, then production build + SW into `dist/` |
| `preview` | `vite preview` | Serve the built output (`:4173`) |
| `icons` | `node scripts/gen-icons.mjs` | Regenerate `public/` PWA icons + OG poster |

**PWA** (`vite.config.ts`): `base:'./'` keeps builds host-agnostic (domain root or
GitHub Pages subpath). `vite-plugin-pwa` `autoUpdate` with Workbox `generateSW`
precaches `js/css/html/png/svg/ico/webmanifest/woff2` (ceiling 4 MB so Phaser's ~1.5 MB
bundle fits), **excludes `og-image.png`** (link-unfurl poster only), and sets
`navigateFallback: index.html`. Manifest: portrait, `standalone`, theme `#f6f3ec`,
192/512/maskable icons.

**Deploy** (`.github/workflows/deploy.yml`): on **push to `main`** (or manual
`workflow_dispatch`), CI checks out, `npm ci`, `npm run build`, uploads `dist/` as a
Pages artifact, and deploys via `actions/deploy-pages@v4` (Node 22, Pages permissions,
`concurrency: pages`). Result is served at the live URL. Install on iOS: Safari → Share
→ Add to Home Screen (offline after first load).
